# Seedance 2.0 Skill OS — Быстрый старт

> Версия 6.6.0 · От установки до первого «срежиссированного» промпта — около 5 минут.
> Полная документация: [README](../README.md).

## В двух словах

Seedance 2.0 Skill OS — это agent skill, который режиссирует Seedance 2.0 как кинематографист, а не нагромождает прилагательные. Правило одно: **режиссируйте модель, а не выстраивайте каждый кадр вручную.** Скажите, что делает сцена, — и skill превратит это намерение в готовый промпт.

## 1. Установка (около 5 минут)

Устанавливайте репозиторий как **одну** корневую skill с именем `seedance-20`; её под-skills и references подтягиваются сами по относительному пути.

**Codex (есть установщик в одну команду)**

```bash
python scripts/install_codex_skill.py --force
```

Копирует репозиторий в `~/.codex/skills/seedance-20` (или `$CODEX_HOME/skills/seedance-20`). Перезапустите Codex и вызовите `$seedance-20`.

**Установка из GitHub (если клиент умеет ставить по URL)**

```text
https://github.com/Emily2040/seedance-2.0
```

**Копирование вручную (другие клиенты)**

Скопируйте папку в каталог skills вашего клиента, не меняя имя `seedance-20`. Типичные пути собраны в [таблице установки в README](../README.md#install) (это не гарантия — проверьте в своём клиенте). Например: Claude Code `.claude/skills/`, Cursor `.cursor/skills/`, GitHub Copilot `.github/skills/`, Windsurf `.windsurf/skills/`.

> Сначала безопасность: ставьте только в те agent-клиенты, которым доверяете. Прежде чем использовать эту skill в стороннем или незнакомом агенте, прочитайте [SECURITY.md](../SECURITY.md).

## 2. Выберите skill под свою задачу

| Что у вас есть… | Загрузить первым |
|---|---|
| пока смутная идея | `seedance-interview` |
| ясная сцена | `seedance-prompt` |
| история из нескольких клипов | `seedance-sequence` |
| принятый клип, который надо продолжить | `seedance-continuation` |
| слабый или заблокированный результат | `seedance-troubleshoot` |
| персонаж, бренд, знаменитость или реальный человек | `seedance-copyright` |

## 3. Сначала режиссура, потом текст — четыре вопроса

1. **Что делает сцена?** Поворот, раскрытие, чувство или демонстрация?
2. **Как об этом говорит камера?** Общий план — про одиночество, крупный — про лицо, наезд — про прозрение.
3. **Ради чего работает свет?** Время суток, жёсткий или мягкий, тёплый или холодный — всё ради намерения.
4. **Что делает звук?** Почти тишина, одна деталь атмосферы или одна реплика.

## 4. Одно сравнение

**Нагромождение (слабо)**

```
эпичный кинематографичный кадр: женщина читает письмо, эмоционально, красивый свет, 4K
```

**Режиссура (сильно)**

```
Средний крупный план, на уровне глаз; она опускает письмо, руки замирают, подступает медленный наезд; мягкий свет из окна оставляет лицо простым; почти тишина, один скрип стула.
```

## 5. Два правила, которые берегут дубли

- **Оставляйте теги ссылок ровно как написано:** `@Image1`, `@Video1`, `@Audio1`, `@图片1`, `@视频1`. Не переводите и не переформатируйте их.
- **Не просите всю историю за одну генерацию.** Сначала сделайте Clip 01, посмотрите, чем он *на самом деле* закончился, и пишите Clip 02 от этого настоящего финала (`seedance-continuation`).

## 6. Безопасность

- **Безопасность контента:** если в идее есть защищённый персонаж, знаменитость, бренд, логотип, песня либо лицо или голос реального человека, не прячьте это на другом языке — перепишите через `seedance-copyright` в оригинальный, лицензированный или постпродакшн-эквивалент.
- **Безопасность агента:** этот пакет **не делает сетевых запросов и не отправляет телеметрию**; его скрипты детерминированы и работают офлайн. Никогда не вставляйте API-ключи, cookie аккаунта или закрытые материалы в агент, которому не доверяете. См. [SECURITY.md](../SECURITY.md).
