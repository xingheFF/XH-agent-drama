## 一、核心原则

相邻镜头之间（无论是否合并到同一视频单元）必须保持视觉逻辑的连贯性。观众在脑中构建的空间关系图（mental map）不能因镜头切换而崩塌。每设计完一个镜头，必须回溯检查与前一个镜头的一致性。

---

## 二、180度轴线规则

### 2.1 基本规则

两个角色的互动场景中，在角色之间画一条虚拟轴线（视线连线），摄影机始终保持在轴线同侧180度范围内。跨越轴线会导致角色左右位置互换，观众混淆。

```
轴线（A←→B）

  摄影机区域（180°）
  \           /
   \    A    /
    \  ↕   /
     \ B  /
      \  /
       \/
   禁止跨越区域
```

### 2.2 角色左右位置映射表

摄影机保持在轴线同侧时：

| 上一个镜头 | 下一个镜头（同侧） | A的位置 | B的位置 |
|-----------|-------------------|---------|---------|
| A在左，B在右，面对镜头 | 拍摄两人背后 | A在右，B在左 | 正确（同侧） |
| A在左，B在右，面对镜头 | 越轴拍摄正面 | A在右，B在左 | 错误（越轴） |
| A在左，B在右 | 过肩拍A（从B肩后） | A仍在中偏左 | 正确 |
| A在左，B在右 | 过肩拍B（从A肩后） | B仍在中偏右 | 正确 |
| A在左，B在右 | POV：A看B | B在画面右侧 | 正确 |
| A在左，B在右 | POV：B看A | A在画面左侧 | 正确 |

### 2.3 越轴的特殊处理

如需越轴，通过以下方式过渡：
- **中性镜头**：插入一个正对轴线的镜头（如正面特写、道具特写），再换到另一侧
- **角色移动**：让角色走动，重新建立轴线
- **摄影机移动**：用一个跟拍/摇镜展示摄影机跨越轴线过程

## 三、视线方向连续性

### 3.1 单人镜头视线方向

| 上一个镜头 | 下一个镜头 | 视线方向约束 |
|-----------|-----------|------------|
| 角色看向画面右侧 | 角色特写 | 眼球仍朝向右侧 |
| 角色抬头仰望 | 俯视POV | POV画面应为角色所看之物 |
| 角色低头 | 角色手中物品特写 | 特写物品方向与低头方向一致 |

### 3.2 双人对话视线方向

| 上一个镜头 | 下一个镜头 | 视线要求 |
|-----------|-----------|---------|
| A在左看向右 | B在右看向左 | 视线交汇，方向匹配 |
| A画面右侧的脸被扇（受力向左） | A偏头向左 | 方向匹配 |
| B从左侧闯入画面 | B的视线向左回看 | 方向统一 |

### 3.3 视线方向设计规范

每个镜头设计时，在「视角」字段后增加一行「视线方向」：
- 单人：明确标注角色视线朝向（画面左/右/上/下/镜头）
- 多人：标注每位角色的视线方向，确认交汇/回避关系
- POV镜头：标注"角色X的POV"，自动确定视线方向

## 四、角色位置映射

### 4.1 位置描述规范

每个镜头的「构图」字段中，必须明确角色在画面中的相对位置：

```
构图：灵汐在画面左侧偏前，章逸在右侧后方，两人面对面
```

### 4.2 相邻镜头位置检查表

| 上一个镜头位置 | 下一个镜头位置 | 是否一致 |
|--------------|--------------|---------|
| A左B右，面对面（正面拍） | A右B左，面对面（背后拍） | ✅ 符合180度规则 |
| A左B右 | A仍左B仍右（过肩拍A） | ✅ 同侧 |
| A左B右 | A右B左（正面拍，未越轴过渡） | ❌ 违反180度规则 |
| A居中 | A居中 | ✅ 单人无轴线问题 |
| A从右侧入画走向左 | A仍在向左移动 | ✅ 方向连续 |
| A从右入画走向左 | 突然A从左侧入画 | ❌ 方向断裂 |

## 五、光线延续

### 5.1 光源方向一致性

同一场景的相邻镜头，主光源方向必须保持一致：

| 上一个镜头光源 | 下一个镜头光源 | 检查 |
|--------------|--------------|------|
| 冷蓝月光从左侧来 | 冷蓝月光从左侧来 | ✅ 一致 |
| 暖黄烛光从右前方 | 暖黄烛光从右前方 | ✅ 一致 |
| 冷蓝月光从左 | 暖黄烛光从左 | ❌ 光色/光质变化需过渡说明 |
| 室外月光（冷蓝） | 室内烛光（暖黄） | ⚠️ 场景切换，需在镜头设计中标明过渡 |

### 5.2 光质连续性

- 硬光→硬光：✅
- 柔光→柔光：✅
- 硬光→柔光：⚠️ 需有过渡（如从室外硬光进入室内柔光，门廊过渡镜头）
- 高光比→低光比：⚠️ 需场景切换或光源变化驱动

### 5.3 光影在角色面上的延续

同一个场景连续镜头，角色面部的光照角度应一致：
- 上一个镜头角色面部左侧明亮右侧阴影 → 下一个镜头同样左侧明亮右侧阴影（同光源方向）
- 若拍摄角色转身/回头，光影在面部应相应变化（如从半明半暗变为全正面光），并在分镜脚本中标注

## 六、动作方向连续性

### 6.1 运动方向一致性

同一动作发生在连续镜头中，运动方向必须一致：

| 上一个镜头 | 下一个镜头 | 连续性检查 |
|-----------|-----------|-----------|
| A从右向左挥手扇出 | 手掌从右击中左脸 | ✅ 方向一致 |
| A从左向右走 | A仍从左向右走 | ✅ 方向一致 |
| A从左向右走 | A从右向左走 | ❌ 方向断裂（除非有转身过渡） |
| 门从右向左推开 | 人从右侧进门 | ✅ 方向匹配 |
| 物体向上抛 | 物体向下落 | ✅ 重力逻辑正确 |

### 6.2 动作速度平滑

相邻镜头中的动作速度不应突变：
- 快速挥动→快速冲击：✅
- 缓慢放下→缓慢停顿：✅
- 快速跑动→突然静止：⚠️ 需有制动/停止的过渡帧

---

## 七、连贯性检查清单

完成分镜脚本设计后，对所有相邻镜头逐一检查以下项目：

```
□ 180度轴线：前后镜头是否在同一侧？
□ 角色位置：A/B的左右关系是否一致或符合映射规则？
□ 视线方向：每位角色的视线方向是否与上一个镜头匹配？
□ 光源方向：主光源方向是否保持一致（同场景）？
□ 光质/光比：是否平滑过渡？
□ 动作方向：运动方向是否连续？
□ 动作速度：速度变化是否有过渡？
□ 场景切换：是否有明确的切换标记和过渡说明？
```

对每个检查项，不通过则修正分镜脚本或在脚本中标注过渡设计。检查结果应内化到分镜脚本的「构图」「光影」「视角」「表演与动作」各字段中，不另起独立清单输出给用户。
