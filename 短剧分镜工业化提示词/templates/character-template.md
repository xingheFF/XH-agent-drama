# 角色档案模板（C001 体系）

> 短剧分镜工业化生产的角色档案标准模板。每个角色分配唯一 `C编号`，存档外貌、服装、性格标签、口头动作等关键信息，确保多镜头生成时角色描述一致。

---

## 一、角色编号规则

| 规则 | 说明 |
|---|---|
| 格式 | `C` + 3 位数字（例：`C001`、`C012`） |
| 分配顺序 | 按角色在剧本中首次出场的顺序 |
| 唯一性 | 一个角色对应一个 `C编号`，不可复用 |
| 跨场稳定 | 同一角色在不同场次的 `C编号` 保持不变 |

---

## 三、档案存储与调用

### 存储方式

- 所有角色档案统一存放于项目根目录的 `characters/` 文件夹下（本技能包不强制创建，用户可自行管理）。
- 文件命名建议：`C001-角色名.md`、`C002-角色名.md`。

### 调用方式

1. **分镜生成时**，AI 根据剧本中出现的角色名，自动匹配 `C编号` 并调用档案中的外貌、服装、性格标签、口头动作、面部表情库。
2. **若档案不存在**，AI 在首次生成时自动创建一个简化版档案（仅含基本信息+外貌特征+服装风格），存入 `characters/` 目录供后续调用。
3. **若用户临时覆盖**（例："C001 这场穿黑色风衣"），AI 在本次生成中使用覆盖值，不修改原档案。

---

## 四、多角色群戏处理

当一场戏出现 3 个以上角色时，按以下规则处理：

1. **主角优先**：分镜中优先描述主角（`C编号` 最小或戏份最重者）的动作与表情。
2. **群像统一**：次要角色统一描述为"多名下属/侍卫/宫女"，不单独分配 `C编号`。
3. **站位图**：复杂群戏建议在分镜卡上方附一张简易站位图（文字描述即可），例：
   ```
   站位图（俯视）：
   案台 ──────── 沈砚(C001)
        │
        李掌柜(C002)
        │
   堂下 ──────── 围观众人
   ```

---

## 五、角色档案维护

- **新增角色**：按模板创建新档案，分配下一个未使用的 `C编号`。
- **修改角色**：直接编辑对应档案文件。修改后需检查已生成的分镜是否需要回溯更新。
- **删除角色**：删除档案文件，并从所有引用该 `C编号` 的分镜中替换为角色名。
- **角色合并**：若发现两个 `C编号` 实为同一角色，保留首次出场的编号，另一个编号作废，相关分镜中的编号同步替换。
